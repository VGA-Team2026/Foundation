# GameReporter

# 概要
- 実行中に様々な処理をするレポータークラス

# 処理
- プログラムで記録開始命令が渡されたら記録を開始する
	- 記録内容はいくつかあり、記録開始時にそれぞれを実行するかどうかフラグで決める
- 記録終了またはアプリケーションの終了をフックとしてレポートを出力する
	- レポートは作業ディレクトリ直下のreportフォルダに出力される
- リリースビルドには含まれない

# 記録内容パターン： リプレイ記録
- ユーザのインプットを時間で記録する
	- `InputEventTrace`を使用して入力値を受け取る
	- ランダム実装があるため、シード値も記録する

# 記録内容パターン： エラーチェック
- 何らかの例外およびエラーログが出力されたかどうかを検知し、そのフレームの情報を出力する
	- エラーログを出力する
	- スクリーンキャプチャを出力する
	- 繰り返し出力されるエラーログの場合は、2回目以降を無視する

# 記録内容パターン： CPUスパイク問題検知
- CPUが動作平均値より著しく低下し、FPS60を下回った場合に問題としてしそのフレームの詳細情報を外部出力する
	- HierarchyFrameDataViewを使用したフレームの記録を行いレポートを出力する
	- スクリーンキャプチャを出力する
- シーン再生(エディタ再生)では`ProfilerRecorder` を使ったリアルタイム監視を行う
	- DrawCall、CPU、GPU、メモリの使用率をグラフ化するためにフレームごとのデータを記録する
		- アロケーションを発生させないよう、5MBほどのバッファを最初に持っておく
	- 記録終了時、またはバッファを使用しきったタイミングで出力する。記録上問題がない場合は出力しない。

# 記録内容パターン： アロケーション問題検知
- GCAlloocが行われていることを監視してレポートする
- シーン再生(エディタ再生)では`ProfilerRecorder` を使ったリアルタイム監視を行う。
	- GCAllocの発生率をグラフ化するために発生フレームのデータを記録して出力する
		- アロケーションを発生させないよう、1MBほどのバッファを最初に持っておく
	- 記録終了時、またはバッファを使用しきったタイミングで出力する。記録上問題がない場合は出力しない。

## 記録内容パターン： 映像記録
- 開始命令が呼ばれた段階からプレイ映像を記録する
	- ビルドでもこの処理が実行できるようにする
- 映像記録の処理設計は以下とする
	- **キャプチャ:** `AsyncGPUReadback` を使用して、GPUのレンダリング結果を非同期でCPUメモリ（NativeArray）に持ってくる（メインスレッドを止めない）。
	    - **転送:** 取得した生データ（Raw Data）を、C#の別スレッド経由で外部プロセス（FFmpeg.exeなど）にパイプライン転送する。
	    - **エンコード:** 別プロセスとして起動したFFmpegが動画（mp4など）にエンコードする。
