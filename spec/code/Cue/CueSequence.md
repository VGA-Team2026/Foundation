# CueSequencerクラス設計

# 概要
- ゲーム中に行われる一連の演出をまとめてシーケンシャルに実行する
- このクラスはツールにより派生クラスが自動生成される

# 必要要件
- 演出の再生管理
- 演出再生に必要な情報群をまとめるクラス構造体

# 実装
- abstract classである
- 抽象化された命令の実装のみを行う
- この派生クラスは再生を行う主体となるクラスが所持し、変数の実体を代入しておく。

# 共通インタフェース(仮想関数)
- PlayAsync: async関数として定義する。CueSequenceを再生する。CueSequenceでの定義は純粋仮想とし、派生先が実装する。

# 派生先のみでの実装
- Setup: 依存関係にある変数の実体を代入する。基底クラスでは実装されない。派生クラスで通常関数として宣言する。

# タイムラインの実装について
- タイムラインのトラックごとに、Asyncで処理される関数を作成し、トラック内の時間までタイマーで待つ実装を行う
	- PlayAsyncで各トラックの処理を実行する関数を呼び出す。これらはすべてasyncで、かつ並列に実行される
	- トラック別の関数の関数名は、トラック名(重複なし)で設定される
	- タイマーはSoundQuantizerのQuantizeTimerを使用する
