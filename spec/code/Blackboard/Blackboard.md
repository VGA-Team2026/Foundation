# Blackboardクラス設計


# 概要
- グローバルに参照されるパラメータを安全に参照するシングルトン


# 実装
- シングルトンでゲーム起動時にクラスは生成される
- 登録＆購読型
- 登録
	- パラメータを全体で参照したい変数にはBlackboardアトリビュートをつける
	- パラメータの大本を使用するクラスで登録関数を呼ぶ。ReactivePropertyを使用する
	- パラメータはカテゴリ名を指定して登録し、Release(カテゴリ名)で一括開放できるようにする
- 購読
	- パラメータを参照したいクラスが、Blackboardのパラメータに購読イベントを設定する。ReactivePropertyを使用する。
	- 購読解除命令を呼び出して、購読が解除できるようにする
- 自動生成
	- アッパーキャメルで値の参照を行えるプロパティ(getのみ)を生成する
		- 名前は大本の変数名と同じにする
		- かぶった場合のみ、クラス名を先頭に付ける
	- 対となる値の登録関数とReactivePropertyを用意する


# 外部インタフェース
- Register: 大本から値の更新を通知する。カテゴリを引数に取る
- Subscribe: 値の更新を購読する
- Describe: 値の購読を解除する
- Release: カテゴリを指定して値の登録と購読を全解除する


# デバッグ
- 登録/購読されている状況をそれぞれ確認できるようにする
- GameManagerに表示用の関数を作成する
