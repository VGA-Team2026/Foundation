# コーディングルール

## 基本要素
- ファイルおよびクラスはなるべく分割する事
- クラス間の関係性は疎結合である事
- テストを書く事で契約的な実装である事を保証する
- データ注入するさいはDIの概念に則る

## ライブラリの利用
- コルーチンはUniTaskを利用する事
- イベント実装はR3を利用する事
- interfaceを使用したクラスをインスペクタに表示するためにSubClassSelectorを使用する
- ObjectPoolはPrefabStockの自前実装を使用する（IObjectPoolインターフェース）

# フォルダ構成
- unity/Assets/Featureフォルダ以下に各機能のコードを格納する
- codeフォルダの階層と同じ階層で配置する

# コメントとコードについて
- Obsidianのリンクの記載は、コメント出力時には無視する
	- ```[[リンク]]```の表記となっているもの

# [デバッグ実装]
- デバッグ実装は、GameManager.IsDebugが有効な際に表示される

# 禁止
- **namespaceはつけないこと**
