# Bashコマンド入力の作法

CLIコマンド実行時のルール。

## ルール

### 1. コマンドを `&&` で繋げない

各コマンドは **個別に** 実行する。
Bashツール呼び出しでも繋げないこと。
許可パターンにマッチしなくなるため。


```
# NG
git add file.cs && git commit -m "msg"

# OK（別々のBash呼び出しで実行）
git add file.cs
git commit -m "msg"
```

依存関係がある場合は順番に呼び出す。独立した場合は並列呼び出し可。

### 2. `cd` でプロジェクトルートに戻る処理は不要

作業ディレクトリは自動的にプロジェクトルートに維持される。
`cd "<project_root>" &&` のようなプレフィックスは不要。

```
# NG
cd "<project_root>" && git status

# OK
git status
```

### 3. まとめ

- 1コマンド = 1 Bash呼び出し
- `cd` プレフィックス不要
- 並列実行可能なコマンドは並列で呼ぶ
