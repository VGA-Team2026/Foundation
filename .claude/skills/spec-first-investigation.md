# 仕様書先行確認スキル

ソースコードの実装調査を行う際、まず仕様書を確認してから実装を読むためのスキル。

## 目的

- 仕様書の記述を先に理解することで、実装の意図を正確に把握する
- 仕様書とソースコードの乖離を発見しやすくする
- 仕様書の存在を忘れずに活用する

## 適用タイミング

以下のケースで仕様書を先に確認する：

1. **Issue対応時**: 対象クラスの実装を読む前に仕様書を確認
2. **バグ調査時**: 問題が発生しているクラスの仕様書を先に確認
3. **機能追加時**: 既存クラスの仕様書を先に確認してから拡張方針を決定
4. **コードレビュー時**: 変更対象クラスの仕様書を先に確認

## 手順

### 1. 仕様書を検索

```
spec/code/ 配下で対象クラス名と同名の .md ファイルを検索
```

仕様書のディレクトリ構造:
- `spec/code/InGame/` - インゲームロジック
- `spec/code/Stage/` - ステージ関連
- `spec/code/Editor/` - エディタ拡張
- `spec/code/System/` - システム基盤
- `spec/code/OutGame/` - アウトゲーム
- `spec/code/Debug/` - デバッグ
- `spec/code/UI/` - UI

### 2. 仕様書を読む

以下を把握する：
- クラスの役割・概要
- プロパティ・フィールド定義
- メソッドの処理フロー
- 期待値・エッジケース
- 注意事項

### 3. 関連仕様書も確認

仕様書内で参照されている関連クラスの仕様書も読む。

### 4. ソースコードを読む

仕様書の理解を前提にソースコードを読み、以下を確認：
- 仕様通りに実装されているか
- 仕様書に記載されていない追加実装があるか
- 仕様書と乖離している箇所があるか

### 5. 乖離がある場合

ソースコードを正として仕様書の更新を提案する。
