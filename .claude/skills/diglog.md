# ログ調査スキル

ゲームログを調査・分析するスキル。ログファイルを検索し、サブエージェントを使って詳細な分析を行う。

## 前提条件

- `unity/gamelog/`ディレクトリにログファイルが存在
- ログファイル形式: `gamelog_{ログコード}.txt`

## ログファイル形式

### ヘッダー
```
================================================================================
Game Log - Code: {ログコード}
Started: {開始時刻}
================================================================================
```

### ログエントリ
```
[HH:mm:ss.fff][ログレベル] [フレーム番号] [メッセージ]
```

- ログレベル: `[INFO]`, `[WARN]`, `[ERROR]`, `[EXCEPTION]`
- フレーム番号: `[数字]` 形式

### リセットマーカー
```
>>> RESET #1 at Frame 1234 <<<
```

### フッター
```
================================================================================
Log Ended: {終了時刻}
Total Reset Count: {リセット回数}
================================================================================
```

## ログファイル検索

### 最新のログファイルを取得
```bash
# Windows (PowerShell)
Get-ChildItem unity\gamelog\gamelog_*.txt | Sort-Object LastWriteTime -Descending | Select-Object -First 1 -ExpandProperty Name

# Unix
ls -t unity/gamelog/gamelog_*.txt | head -1
```

### 特定コードのログを検索
```bash
ls unity/gamelog/gamelog_*{ログコード}*.txt
```

## 調査フロー

1. **仕様書検索によるコード特定**
   - 解析内容のキーワードから `spec/code/` 配下を検索
   - 関連クラス・メソッド・ログ出力パターンを特定
   - `spec/ubi/` のユビキタス言語辞書も必要に応じて参照
   - 特定したクラス名・メソッド名をログ検索キーワードに追加

2. **ログファイルの特定**
   - ログコードが指定されていれば該当ファイルを検索
   - 指定されていなければ最新ログを使用

3. **基本情報の確認**
   - ログコード
   - 開始・終了時刻
   - 総リセット回数
   - ファイルサイズ

4. **サブエージェントの起動**
   - Sonnetを使用したlog-investigatorを起動
   - 仕様書検索で特定したコード情報も含めて指示

5. **動画フレーム分析（必要に応じて）**
   - ログ調査で問題フレームが特定された場合
   - ffmpegで該当時刻のフレーム画像を抽出
   - Readツールで画像を読み込み視覚的に分析
   - フレーム番号→動画秒数の変換: `経過秒 ≈ フレーム番号 / 60`

6. **レポート作成**
   - 調査結果をマークダウン形式でまとめる
   - 関連コード情報・フレーム分析結果も含める

## サブエージェント起動例

### 仕様書検索エージェント
```
Task(subagent_type="Explore", model="haiku", prompt="
spec/code/ 配下の仕様書から「{キーワード}」に関連する
クラス・メソッド・ログ出力を検索してください。

報告内容:
- 関連クラス名とファイルパス
- 関連メソッド名
- ログ出力に使われるタグやメッセージパターン
- 関連するソースコードのパス
")
```

### ログ調査エージェント
```
Task(subagent_type="general-purpose", model="sonnet", prompt="
ログファイルを分析してください。

## ログファイル
{ログファイルパス}

## 解析内容
{解析内容}

## 関連コード情報（仕様書検索結果）
{特定したクラス名・メソッド名・ログパターン}

## 調査範囲
{フレーム数またはステージブロック名}

## 調査手順
1. ログファイルを読み込む
2. 関連コード情報のクラス名・タグでログエントリをフィルタリング
3. 解析内容に関連するログエントリを抽出
4. 時系列で分析
5. 問題の原因や経緯を特定
6. 結果をレポート形式でまとめる

## 出力形式
### 調査結果サマリ
- 発見した問題点や注目すべきイベント

### 詳細分析
- 関連するログエントリの抜粋と解説

### 考察
- 原因の推測や対応案
")
```

## 動画フレーム分析

### ffmpegコマンド例
```bash
# 特定時刻のフレーム抽出
ffmpeg -ss {秒} -i {動画} -frames:v 1 -q:v 2 {出力先}/frame_{フレーム番号}.png

# 問題箇所前後の連続抽出（前後1秒、5fps）
ffmpeg -ss {開始秒} -t 2 -i {動画} -vf fps=5 -q:v 2 {出力先}/frame_%04d.png
```

### 動画ファイルの検索先
- `log/unitylog/*{ログコード}*.mp4`
- `log/unitylog/*{ログコード}*.mkv`
- `unity/gamelog/*{ログコード}*.mp4`
- 引数で明示的に指定されたパス

## 検索パターン

### フレーム数での絞り込み
- 単一フレーム: `1000` → フレーム1000周辺
- 範囲指定: `1000-2000` → フレーム1000〜2000

### ステージブロック名での絞り込み
- ステージブロック名（例: `StageLevel_4-10`）でログをフィルタリング
- カーブマップ設定: `CurveMap setup complete for {ステージ名}`

### よく使う検索キーワード
- プレイヤー死亡: `[Player]`, `Die`, `GameOver`
- カーブ: `Curve`, `CurveMap`, `ProcessCurveInput`
- 敵: `[Enemy]`, `Attack`, `Octopus`, `Usagi`, `Kuronoir`
- 衝突: `OnHit`, `Collision`, `HitDetector`
- ステージ: `StagePresenter`, `StageBehaviour`, `StageCell`

## レポート形式

```markdown
# ログ調査レポート

## 調査情報
- ログコード: {ログコード}
- ログファイル: {ファイルパス}
- 解析内容: {解析内容}
- 調査範囲: {フレーム数/ステージブロック名}

## 関連コード
- {仕様書検索で特定したクラス名・メソッド名とソースパス}

## 調査結果
{サブエージェントからの結果}

## フレーム分析（動画がある場合）
- 問題フレーム: {フレーム番号} ({経過秒数}s)
- 画像所見: {抽出画像から確認できた状況}

## 結論
{分析に基づく結論}
```
