# ログ調査コマンド

ゲームログを調査・分析するコマンド。指定されたログコードに対応するログファイルを検索し、解析内容に基づいてログを調査する。

## ロール
あなたはゲームプログラマです。
ゲームログを分析し、問題の原因特定や動作確認を行います。

## モデル使用方針
- **分析・判断**: メインエージェントで行う
- **ログ調査**: Sonnetを使用したサブエージェント（log-investigator）を起動して行う
- サブエージェントには具体的な調査タスクを指示し、結果を受け取って分析する

## 引数
コマンド引数には以下の形式で渡される:
```
/diglog [ログコード] [解析内容] [フレーム数またはステージブロック名] [動画パス]
```

- **ログコード**: 英数字のログ識別コード（省略可能、省略時は最新のログを使用）
- **解析内容**: 調査したい内容（必須）
- **フレーム数またはステージブロック名**: 調査範囲の絞り込み（省略可能）
- **動画パス**: 分析対象の動画ファイルパス（省略可能、省略時はログコードで自動検索）

### 使用例
```
/diglog 0124155355 カーブ失敗の原因
/diglog カーブ失敗の原因 1000-2000
/diglog abc123 プレイヤー死亡 StageLevel_4-10
/diglog 敵の攻撃タイミング
/diglog 0124155355 カーブ失敗の原因 1000-2000 replay.mp4
```

## 処理フロー

### 1. 引数の解析
- 引数がない場合は使用方法を表示
- 最初の引数が英数字のみ（8-10桁程度）の場合はログコードとして扱う
- ログコードが指定されていない場合は最新のログファイルを使用

### 2. 仕様書検索によるコード特定
解析内容のキーワードから、関連する仕様書とソースコードを特定する。

**手順:**
1. 解析内容からキーワードを抽出（例: 「カーブ失敗」→「カーブ」「Curve」）
2. `spec/code/` 配下の仕様書を検索してクラス・メソッドを特定
3. 必要に応じて `spec/ubi/` のユビキタス言語辞書も参照
4. 特定したクラス名・メソッド名をログ検索キーワードに追加

```
Task(subagent_type="Explore", model="haiku", prompt="
spec/code/ 配下の仕様書から「{解析内容のキーワード}」に関連する
クラス・メソッド・ログ出力を検索してください。

以下を報告してください:
- 関連クラス名とファイルパス
- 関連メソッド名
- ログ出力に使われるタグやメッセージパターン
- 関連するソースコードのパス
")
```

### 3. ログファイルの検索
```bash
# ログファイルの一覧を取得（日時降順）
ls -lt unity/gamelog/gamelog_*.txt | head -10
```

ログファイルの命名規則:
- 形式: `gamelog_{ログコード}.txt`
- 例: `gamelog_0124155355.txt`

指定されたログコードと一致するファイルを検索:
```bash
ls unity/gamelog/gamelog_*{ログコード}*.txt
```

### 4. ログファイルの読み込み
ログファイルを読み込み、基本情報を確認:
- ログコード
- 開始時刻
- 総リセット回数
- ファイルサイズ

### 5. ログ調査エージェントの起動
Taskツールでlog-investigatorサブエージェント（Sonnet）を起動:

```
Task(subagent_type="general-purpose", model="sonnet", prompt="
ログファイルを分析してください。

## ログファイル
{ログファイルパス}

## 解析内容
{解析内容}

## 関連コード情報（仕様書検索結果）
{Step 2で特定したクラス名・メソッド名・ログパターン}

## 調査範囲
{フレーム数またはステージブロック名}

## 調査手順
1. ログファイルを読み込む
2. 関連コード情報のクラス名・タグでログエントリをフィルタリング
3. 解析内容に関連するログエントリを抽出
4. 時系列で分析
5. 問題の原因や経緯を特定
6. 結果をレポート形式でまとめる

## 出力形式
### 調査結果サマリ
- 発見した問題点や注目すべきイベント

### 詳細分析
- 関連するログエントリの抜粋と解説

### 考察
- 原因の推測や対応案
")
```

### 6. 動画フレーム分析（必要に応じて）
ログ調査で問題フレームが特定された場合、対応する動画があればffmpegで画像を抽出して視覚的に分析する。

**前提条件:**
- 動画ファイルが存在すること（引数で指定 or ログコードに対応する動画を自動検索）
- ffmpegがインストール済みであること

**動画ファイルの検索:**
```bash
# ログコードに対応する動画を検索
ls unity/gamelog/*{ログコード}*.mp4 unity/gamelog/*{ログコード}*.mkv 2>/dev/null
# Melpomene録画フォルダも検索
ls unity/gamelog/*{ログコード}*.mp4 2>/dev/null
```

**フレーム画像の抽出:**
```bash
# 特定の時刻（秒）のフレームを抽出
ffmpeg -ss {問題発生時刻（秒）} -i {動画ファイル} -frames:v 1 -q:v 2 {出力先}/frame_{フレーム番号}.png

# 問題箇所前後のフレームを連続抽出（前後1秒、5fps）
ffmpeg -ss {開始秒} -t 2 -i {動画ファイル} -vf fps=5 -q:v 2 {出力先}/frame_%04d.png
```

**出力先:** スクラッチパッドディレクトリを使用

**ログの時刻→動画の時刻への変換:**
- ログのゲーム開始時刻と動画の開始時刻を照合
- フレーム番号からおおよその経過秒数を算出: `経過秒 ≈ フレーム番号 / 60`（60fps想定）
- リセットマーカーに注意（リセット後はフレーム番号がリセットされる）

**画像分析:**
- 抽出したフレーム画像をReadツールで読み込み、視覚的に分析
- プレイヤー位置、障害物配置、エフェクト状態などを確認
- 問題の状況を画像から裏付ける

### 7. レポートの出力
調査結果を以下の形式で出力:

```markdown
# ログ調査レポート

## 調査情報
- ログコード: {ログコード}
- ログファイル: {ファイルパス}
- 解析内容: {解析内容}
- 調査範囲: {フレーム数/ステージブロック名}

## 関連コード
- {仕様書検索で特定したクラス名・メソッド名とソースパス}

## 調査結果
{サブエージェントからの結果}

## フレーム分析（動画がある場合）
- 問題フレーム: {フレーム番号} ({経過秒数}s)
- 画像所見: {抽出画像から確認できた状況}

## 結論
{分析に基づく結論}
```

## ログファイル形式

### ヘッダー
```
================================================================================
Game Log - Code: {ログコード}
Started: {開始時刻}
================================================================================
```

### ログエントリ
```
[HH:mm:ss.fff][ログレベル] [フレーム番号] [メッセージ]
```

- ログレベル: `[INFO]`, `[WARN]`, `[ERROR]`, `[EXCEPTION]`
- フレーム番号: `[数字]` 形式

### リセットマーカー
```
>>> RESET #1 at Frame 1234 <<<
```

### フッター
```
================================================================================
Log Ended: {終了時刻}
Total Reset Count: {リセット回数}
================================================================================
```

## 検索パターン

### フレーム数での絞り込み
- 単一フレーム: `1000` → フレーム1000周辺
- 範囲指定: `1000-2000` → フレーム1000〜2000

### ステージブロック名での絞り込み
- ステージブロック名（例: `StageLevel_4-10`）でログをフィルタリング
- カーブマップ設定: `CurveMap setup complete for {ステージ名}`

### よく使う検索キーワード
- プレイヤー死亡: `[Player]`, `Die`, `GameOver`
- カーブ: `Curve`, `CurveMap`, `ProcessCurveInput`
- 敵: `[Enemy]`, `Attack`, `Octopus`, `Usagi`, `Kuronoir`
- 衝突: `OnHit`, `Collision`, `HitDetector`
- ステージ: `StagePresenter`, `StageBehaviour`, `StageCell`

## 注意事項
- ログファイルが大きい場合は、フレーム数やステージブロック名で範囲を絞り込むこと
- リセットマーカーに注意して、どのプレイセッションのログかを確認すること
- エラー・警告ログは特に注目すること
